# NVIDIA H100 SXM — Hopper (GH100)

Production optimization target. Faster than L40S but not KernelBench-comparable.

## Compute
- SM count: 132
- FP32: 67 TFLOPS
- FP16: 1,979 TFLOPS (with tensor cores)
- BF16: 1,979 TFLOPS (with tensor cores)
- TF32: 989 TFLOPS (with tensor cores)
- FP8: 3,958 TOPS (with tensor cores)
- Clock: 1830 MHz boost

## Memory
- VRAM: 80 GB HBM3
- Bandwidth: 3,350 GB/s
- L2 cache: 50 MB
- L1/shared per SM: 256 KB (configurable split)
- Register file per SM: 256 KB

## Tensor Cores
- 4th gen tensor cores (Hopper)
- TMA (Tensor Memory Accelerator) for async bulk copies
- Warp group MMA: 64x128x16 (fp16), supports fp8
- Thread block clusters for cross-SM cooperation

## Key Ratios for Optimization
- Arithmetic intensity crossover (fp32): ~20 FLOP/byte
- Arithmetic intensity crossover (fp16): ~591 FLOP/byte
- HBM3 has much higher bandwidth than GDDR6X — memory-bound threshold is different

## Tuning Guidance
- Larger tiles preferred: 128x256 or 256x128 for GEMM
- num_stages: 3-4 (HBM3 latency is lower than GDDR6X)
- TMA descriptors via Triton's block pointer API for best performance
- Persistent kernels benefit from thread block clusters
