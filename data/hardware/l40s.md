# NVIDIA L40S — Ada Lovelace (AD102)

KernelBench official evaluation hardware. All published baselines report on L40S.

## Compute
- SM count: 142
- FP32: 91.6 TFLOPS
- FP16: 733 TFLOPS (with tensor cores)
- BF16: 733 TFLOPS (with tensor cores)
- TF32: 366 TFLOPS (with tensor cores)
- INT8: 1,466 TOPS (with tensor cores)
- Clock: 2520 MHz boost

## Memory
- VRAM: 48 GB GDDR6X (not HBM)
- Bandwidth: 864 GB/s
- L2 cache: 96 MB
- L1/shared per SM: 128 KB (configurable split)
- Register file per SM: 256 KB (65536 x 32-bit)

## Tensor Cores
- 4th gen tensor cores (Ada)
- Warp-level MMA: 16x8x16 (fp16), 16x8x8 (tf32)
- Triton uses tl.dot → tensor core MMA automatically

## Key Ratios for Optimization
- Arithmetic intensity crossover (fp32): ~106 FLOP/byte
- Arithmetic intensity crossover (fp16): ~849 FLOP/byte
- Ops/byte ratio tells you memory-bound vs compute-bound:
  - matmul 4096x4096: ~2730 FLOP/byte → compute-bound
  - elementwise (add, relu): ~0.25 FLOP/byte → memory-bound
  - softmax: ~5-10 FLOP/byte → memory-bound
  - layernorm: ~10-20 FLOP/byte → memory-bound
  - conv2d: varies by filter size, usually compute-bound for 3x3+

## Occupancy Guidelines
- Max threads per SM: 1536
- Max blocks per SM: 16 (Ada)
- Shared memory per SM: 100 KB (when L1 is minimized)
- Target occupancy: 50%+ for latency hiding
- More important for memory-bound kernels than compute-bound

## Tuning Guidance
- BLOCK sizes for tiling: try {64, 128, 256} for M/N, {32, 64} for K
- num_warps: 4 for small tiles, 8 for 128x128+
- num_stages: 3-5 for software pipelining (async loads)
- GDDR6X has higher latency than HBM — pipelining matters more
